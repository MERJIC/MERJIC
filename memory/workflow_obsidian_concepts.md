# 概念知识库工作流

## 背景

麦橘是哲学本科背景，曾有完整的知识体系，因时间久远和方向转型而弱化。现在以 Obsidian 为载体重新建设，面向长期积累（10,000 条概念量级）。他是长期主义者，这不是短期项目——在每次寓言故事 / 圆桌讨论时，都应意识到这是在为长期知识网络添加节点和边，维护质量和一致性比速度更重要。

## 设计原则

奥卡姆剃刀：维护成本必须接近零。任何需要人工同步两处以上的设计，时间长了都会腐烂。这套设计应当能扩展到 10,000 条概念，写第 8,000 条的成本和写第 1 条基本相同。

## 知识来源与闭环路径

**被动学习**（由外部递送）：`/寓言故事`、圆桌衍生概念

**主动学习**（麦橘带着问题来）：外部阅读（`source: reading`）、随机对话（`source: conversation`）

两种来源的区别体现在**入口场景**的写法：被动学习写故事关键情节，主动学习写麦橘当时的问题意识。几年后翻看概念页，这条线索比原理本身更有价值。

```
/寓言故事 [领域]  →  故事  →  概念页（建节点）
        ↓（可选）
/roundtable             →  圆桌  →  沉淀回概念页（建边，增加 [[]] 链接）
                                   ↓
                              衍生新概念 → 新概念页 + 虚链接
```

圆桌的角色是**建边**，不是建节点。一次圆桌结束，知识网络里增加的主要是连接，而不是新词条。

## 虚链接 = 待建概念队列

正文中自然提到某个概念时，直接写 `[[概念名]]`，不管它是否已有页面。已有页面则形成双向链接；尚未建页则形成虚链接（空节点）。虚链接不是错误，是信号——被高频引用的空节点就是下一个值得展开的概念。我会主动提出"要建页吗"，不等麦橘来问。

**主动学习入口（麦橘的习惯）：** 在 Obsidian 内阅读概念页时，遇到陌生词直接划词新建 stub 页（Obsidian 原生功能）。之后来找我补充内容。这是比规划式建库更自然的路径，应当支持而不是干预。

## 链接密度的预期管理

**现阶段（<100 条）：** 不追求图谱密度。强行补链的联系没有信息量，意义不大。知识图谱的涌现发现和顺藤摸瓜导航，在几百条量级之后才有实际价值。

**链接的产生方式：** 只允许有机生长——写作时自然提到，或麦橘读页面时划词新建。不做回溯式强制补链，也不主动建议麦橘去补。

## 概念页规范

### 字段（5个，固定）

```yaml

domain: [哲学]               ← 顶层学科，只能从固定词汇表 9 个值里选

```

禁用字段：`name_en` `title` `slug` `created` `related` 等一律不用。

### 正文结构

**必有：** `## 核心机制`、`## 入口场景`

**可选：** `## 现实锚点`、`## 适用边界`、`## 圆桌沉淀`

**删除：** `## 关联概念` section，关联改为正文自然 [[]] 链接。

### 格式自检（写完立即做，不等麦橘指出）

1. frontmatter 字段名全部在模板列表内，无禁用字段
2. `tags` 值全部为 `pattern/`、`apply/` 或 `discipline/` 开头，且在固定词汇表内
3. `domain` 是数组格式，学科名在顶层 9 个值内

## 工作流行为规则

### 寓言故事

- 麦橘说「讲个寓言故事」「讲寓言」「讲个故事」时，立即调用 `Skill({skill: "parable-story"})`，不直接输出故事
- 故事输出后**不得自动创建概念页**，必须等麦橘明确确认（「建页」「存」「OK」或类似指示）后再写入
- Why: skill 目录名含中文时曾加载失败；故事输出后信息可能不完备，需麦橘审核后再沉淀

### 圆桌讨论

- 圆桌结束后**自动**把讨论沉淀写回对应概念页的「圆桌沉淀」section，无需麦橘另行提醒
- 沉淀内容**尽可能完整**：保留日期、嘉宾阵容、核心争议的完整论述、各方观点、留存洞见，不做摘要式压缩
- 所有出现过的概念名用 `[[]]` 标注
- 有值得独立成页的衍生概念时主动提出，等确认后写入
- Why: 麦橘明确要求概念页内容尽可能详细，圆桌里的论证、争议、洞见都是知识的一部分

### 概念卡片

- 配色固定：米白底 `#F4EFE4` + 橘 `#E66A2C`，浅/深交替节奏（`orange-card-style.css`）
- 圆角固定：`border-radius: 0`（小红书自带圆角，无需重复）
- 水印固定：`MERJIC`（ZY LOYALTY 字体），不写「麦橘」
- 排版自由：封面/金句/引文/数字编号/留白问句等页型按内容选择，不强行统一，每张卡只承担一个信息功能
- Why: 麦橘明确说不同内容有不同呈现方式，固定排版会让所有内容看起来一样

### 项目级变更

- 对项目做全局性变更（改 skill 名称、调整工作流、修改规范）时，**必须同时更新 CLAUDE.md**，不能只改 MEMORY.md
- Why: CLAUDE.md 是每次对话真正生效的指令文件，MEMORY.md 只是索引，只改 MEMORY.md 等于变更没有落地

**组织哲学：** 链接优先于目录 / 标签辅助检索 / 扁平 + 索引 / 属性即元数据 / 反向链接是导航

## 长期维护职责

1. 圆桌结束后自动沉淀，沉淀时把所有概念名用 [[]] 标注
2. 高频被引用的虚链接，主动提出「要建页吗」
3. 新学科名或新 apply 场景出现时，先更新词汇表再使用
4. 写完概念页立即执行格式自检三条，不等麦橘开口
5. 旧概念页格式不一致时，在 consolidate-memory 时一并修正
