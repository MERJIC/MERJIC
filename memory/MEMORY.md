## 概念页字段规范

**5 个固定字段，不多不少：**

```yaml
name: 弱意志（Akrasia）      ← 中英文写在一起
domain: [哲学]               ← 顶层学科，只从下方 9 个值选
date: 2026-05-08
source: 寓言故事
tags: [discipline/伦理学, pattern/冲突, apply/自我]
```

禁用：`name_en` `title` `slug` `created` `related`

**正文必有：** `## 核心机制`、`## 入口场景`

**写完立即自检（不等麦橘开口）：**
1. frontmatter 字段名全部在模板列表内，无禁用字段
2. `tags` 值全部为 `pattern/`、`apply/` 或 `discipline/` 开头，且在固定词汇表内
3. `domain` 是数组格式，学科名在顶层 9 个值内

## 行为规则

**寓言故事**
- 收到「讲寓言/讲故事」指令，立即调用 `Skill({skill: "parable-story"})`，不直接输出
- 故事输出后不得自动建页，等麦橘明确确认（「建页」「存」「OK」）后再写入
- 建页前先检查库里是否已有该概念
- 建页前读取 `memory/concept_relations.md`：确认新概念的集群归属，在正文中自然链接集群内相关概念；检查该概念是否在孤立列表中，若在则标记为"新增出链"

**圆桌讨论**
- 结束后自动把讨论沉淀写回对应概念页的 `## 圆桌沉淀` section，无需麦橘提醒
- 沉淀内容保留完整：日期、嘉宾、核心争议、各方观点、洞见，不做摘要压缩
- 出现的概念名全部用 `[[]]` 标注；有值得独立建页的衍生概念，主动提出

**概念卡片**
- 配色：米白底 `#F4EFE4` + 橘 `#E66A2C`
- 水印：`MERJIC`（ZY LOYALTY 字体），不写「麦橘」
- 圆角：`border-radius: 0`

**项目级变更**
- 对工作流、规范、skill 做全局性变更时，必须同时更新 workflow_obsidian_concepts.md
