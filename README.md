# 麦橘概念库

125 个跨领域概念的结构化笔记，覆盖哲学、心理学、行为经济学、社会学等 9 个学科。持续增加中。

每个概念有固定格式：**核心机制 + 入口场景**，加上现实锚点和适用边界。概念之间通过 `[[wikilink]]` 相互引用，形成有机生长的知识网络。

---

## 这个库能做什么

**直接阅读**。每个概念是一个独立 Markdown 文件，GitHub 上直接可读，不需要任何工具。[概念索引](概念页/INDEX.md) 按学科分组，125 个概念全部可点击跳转。

**跨领域结构模式检索**。`pattern/` 标签揭示不同学科里结构相同的概念。经济学的「双曲折现」、传播学的「沉默的螺旋」、心理学的「弱意志」，底层都是同一个结构：`pattern/循环`——当下的选择不断推翻长期的判断。用结构而非学科来理解世界，视角会变。

**按实际问题反查**。`apply/` 标签对应现实场景。面对组织决策失效，检索 `apply/组织`；理解关系里的沉默，检索 `apply/关系`。概念不是用来收藏的。

**辨析近似概念**。每个页面的「适用边界」专门处理容易混淆的概念对：沉默的螺旋 vs 多元无知 vs 偏好伪装，团体迷思 vs 阿比林悖论，道德风险 vs 逆向选择。很多时候，搞清楚两个相似概念的边界，比理解每个概念本身更有用。

**入口场景方法本身**。每个概念用一个具体情境打开——不是定义，是一个让概念「亮」起来的场景。这个写法本身是一种可以借鉴的思维和表达方式。

## 概念格式

每个概念页的结构固定，5 个 frontmatter 字段：

```yaml
---
name: 古德哈特定律（Goodhart's Law）
domain: [管理学, 经济学]
date: 2026-05-13
source: 对话整理
tags: [pattern/错位, apply/组织]
---
```

正文必有两节：

- **核心机制** — 概念的原理、来源和本质
- **入口场景** — 一个具体场景，让抽象的东西有落脚点

常见可选节：

- **现实锚点** — 概念在不同语境下的多个案例
- **适用边界** — 概念的适用限制、与近似概念的辨析

## 示例概念页

以下是一个完整概念页的内容：

---

**古德哈特定律（Goodhart's Law）**

`domain: 管理学, 经济学` · `pattern: 错位` · `apply: 组织`

**核心机制**

「当一个指标变成目标，它就不再是好指标。」

一旦某个指标与奖惩挂钩，人们就会直接优化指标本身，而不是指标背后试图测量的真实目标——两者开始分叉。

**入口场景**

客服团队考核「工单关闭数量」，客服开始快速关闭工单；遇到复杂问题就拆成多张。管理层看到数量上升以为效率提高，实际上客户满意度在悄悄下滑。

**现实锚点**

- 教育：标准化考试分数作为质量代理指标 → 学校开始应试教学，分数上去了，实际能力没跟上
- 软件：用代码行数衡量工程师产出 → 提交变碎、代码变冗余
- 销售：只考核成交量不考核留存 → 拼命成交低质客户，流失率飙升

越容易量化的事，越容易被优化成空壳。应对方向不是「找到更好的指标」——新指标很快也会被 Goodhart——而是多指标组合 + 定性校验。

**适用边界**

委托代理问题中尤为突出；[[帝国建造]] 是典型例证。

## 使用

**在 GitHub 上直接读**：打开 [`概念页/INDEX.md`](概念页/INDEX.md)，支持按学科、按结构模式、按应用场景三种方式浏览，全部可点击跳转。

**用 Obsidian 打开**（获得图谱视图和双向链接）：

```bash
git clone https://github.com/MERJIC/noosphere.git
```

用 Obsidian 打开克隆下来的 `概念页/` 文件夹作为 Vault，可使用双向链接和关系图功能。

**参与**：发现错误、有更好的入口场景、或想建议新概念，欢迎开 Issue。

## 关于

这是一个长期知识积累项目。每个概念的「入口场景」是关键——不是定义罗列，而是找一个让这个概念「亮」起来的具体情境。

设计目标是维护成本接近零，能扩展到数千条概念，写第 500 条和写第 1 条的结构成本基本相同。
