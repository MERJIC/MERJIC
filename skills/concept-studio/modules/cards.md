## 设计原则

以下原则是这套卡片系统的根基。参数会随版本迭代，原则不变。

### P1. 底色有节奏

深色和浅色交替出现，视觉上产生呼吸感。特殊卡型（综述、封面、结尾）用强调色或固定深色，打断交替节奏形成锚点。

原则：深浅交替，关键位打破。不是机械轮换，而是有律动的编排。

### P2. 底色决定文字色

深底白字，浅底黑字。不存在「深底灰字」或「浅底白字」这种读者看不清的组合。

原则：可读性是底线，底色和文字色的对比度必须充足。

### P3. 人物有颜色身份

多位参与者出现时，每人分配一个固定颜色。这个颜色贯穿他出现的所有位置——圆点、竖线、高亮词。

原则：颜色是人物的视觉签名，不随底色变化而丢失，只做明暗适配。

### P4. 封面是钩子

封面不写人名、不写目录、不写摘要。用一句直击核心困惑的话让读者想翻下去。

原则：小红书是信息流，封面只有一秒的窗口——一句话钩子，不多废话。

### P5. 品牌水印低调

右下角 MERJIC，字体 ZY LOYALTY，小到不影响阅读，但能辨认。

原则：品牌存在感来自内容质量，不来自水印大小。

### P6. 内容不妥协

传播版和留档版的区别在表达方式，不在内容深度。专业术语保留，读者能跟上逻辑。

原则：高认知读者看出深度，普通读者跟上逻辑，两者不互斥。

### P7. 排版为内容服务

没有 ASCII 图表（小卡片上排版不可控）、没有花哨装饰、没有无关元素。直角、无阴影、无渐变。

原则：每张卡只有一个焦点——内容本身。

### P8. 不套格式，做判断

本文件记录的是经过验证的参数和规则，不是必须填的表。内容不同，卡片数量、底色节奏、结构编排都应该灵活调整。

原则：理解规则背后的「为什么」，比遵守规则本身更重要。

### 底色编排规则

- 只有三种底色，禁止 grey 或其他中间色
- 综述卡用 accent，封面和结尾用 dark
- 非固定位严格深浅交替

### 人物颜色默认色板

可按人物气质调整，但五色必须和谐且有区分度：


| 位置        | 色1        | 色2        | 色3         | 色4        | 色5        |
| --------- | --------- | --------- | ---------- | --------- | --------- |
| 默认        | #e0620a 橙 | #c0392b 红 | #f1c40f 金黄 | #3498db 蓝 | #27ae60 绿 |
| dark 提亮   | —         | —         | #d4a500    | —         | —         |
| accent 降级 | 白         | 白         | 白          | 白         | 白         |


颜色出现在：阵容卡圆点、speaker-dot、quote-block 竖线、正文高亮词。

### 人名格式

- 阵容卡：`中文全名  英文全名`（双空格，无 `·`）
- 发言卡/对话卡：`中文简称 · 英文简称`
- 发言正文：纯中文简称，不出英文

### 品牌水印

- 字体 ZY LOYALTY，字号 9px，字间距 0.06em
- 只写 `MERJIC`，无其他文字

### 人物选择

- 不限于 scholar-dict 中的学者
- 可加入 influencer、公众人物、行业实践者
- scholar-dict 仅用于名字拼写对照

### 排版硬约束

- 固定宽高 375×500px，不用 aspect-ratio
- 每张卡 2-3 段正文
- 禁止 ASCII 图表、frame-box、Noto Serif SC
- 直角无阴影无圆角无渐变

## 内容改写规范

### 专业词汇

- 保留，第一次出现时紧跟一句话解释，后续直接使用

### 叙述视角

- 人物发言用第一人称，主持/综述用第三人称或无人称
- 一张卡片内不混用

### 禁用结构

- 否定排比、公式化结尾、学术标注（"核心命题""核心挑战"）
- 简言之/总而言之总结框——核心观点融入正文
- 论点条目化——还原成发言现场，不要列表

### 圆桌语气分化

- 每位发言者有不同的语气节奏
- 第一人称直接说立场，不用"我认为""我觉得"缓冲

## 技术规范

### HTML 生成

将 CSS 内联到 `<style>` 标签（不用外链，保证离线可用和导出完整）。

### 导出方案

**必须用浏览器原生 `window.print()`。禁止使用 html2canvas。**

html2canvas 在深色背景上无法正确渲染白色文字（已验证为不可修复 bug）。浏览器原生打印走 Chrome 自身渲染引擎，颜色和字体 100% 精确。

必须包含 `@media print` CSS 规则：

```css
@media print {
  @page {
    size: 375px 500px;
    margin: 0;
  }
  body { margin: 0; padding: 0; background: none; }
  .page-header, .hint, .export-bar { display: none !important; }
  .stack { gap: 0 !important; padding: 0 !important; }
  .card {
    page-break-after: always;
    break-after: page;
    margin: 0 !important;
    border: none !important;
    box-shadow: none !important;
    print-color-adjust: exact;
    -webkit-print-color-adjust: exact;
  }
  .card:last-child { page-break-after: avoid; break-after: avoid; }
}
```

`.card` 上必须有 `print-color-adjust: exact; -webkit-print-color-adjust: exact`，否则打印时背景色消失。

### 生成流程

1. 确定使用哪个版式
2. 读取对应的 DESIGN.md（不是 cards.md 里的参数）
3. 从 DESIGN.md 的设计语言出发编排卡片，不套其他版式的原则
4. 内容（圆桌讨论文本）不变，呈现方式随版式变化
5. 导出统一用 window.print()
