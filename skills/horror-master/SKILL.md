## 模块二：编剧诊断

用户给出场景描述、角色设计、剧情片段或剧本草稿，扫描其中认知机制的覆盖度，找出盲区。

**诊断不直接给写法建议——那是模块二B的事。** 诊断回答的是：你的素材里哪些认知机制在起作用、哪些没用到、哪些用到了但深度不够。

### 诊断流程

**Step 1：理解素材**

确认用户提供的是什么——场景？角色？剧情线？完整剧本片段？问清楚再动手。

**Step 2：概念扫描**

从十三条脉络中扫描与素材相关的概念。判断标准：
- 场景涉及什么空间？→ 阈限性、全景监狱、氛围、废墟美学
- 角色面临什么威胁？→ 入侵类（寄生/拟态/附身）、边界类（侵犯/创伤/双重束缚）、认知类（归因失效/叙述者可疑/认知失调）
- 观众被放在什么信息位置？→ 悬念与惊吓、测不准、缺笔、间离效果
- 恐惧的尺度？→ 个体（身体图式、否认、焦虑）、人际（边界侵犯、凝视、间性）、群体（情感传染、临界质量、团体迷思）、社会（平庸之恶、例外状态、认识论不正义）
- 恐惧的维度？→ 认知（归因失效/范畴错误）、身体（贱斥/物化/具身认知）、伦理（道德创伤/道德分离/道德运气）、美学（氛围/消逝感/废墟美学）、政治（景观社会/内化压迫/象征性暴力）
- 结局走向？→ 压抑回归（闭合）、开放结局（不闭合）、崇高（超越）、向死而生（本真地活）

**Step 2.5：确认诊断方向**

列出识别到的概念脉络，问用户：「我准备从这几个方向诊断，需要调整吗？」用户确认后再进入 Step 3。如果用户补充了新信息，回到 Step 2 重新扫描。

**Step 3：诊断输出**

输出结构：

```
## 诊断结果

### 已激活的认知机制
- [脉络名]：[素材中的具体体现] [深度：浅/中/深]

### 未覆盖的盲区
- [脉络名]：[这条脉络的概念在你的素材里完全没出现，可能的原因]

### 覆盖了但深度不够
- [脉络名]：[表面触及了但可以推得更深的方向]

### 诊断总结
- 最强的一根线：[脉络名]，因为[原因]
- 最大的缺口：[脉络名]，因为[原因]
- 下一步建议：走模块二B（编剧操作）或直接走剧本生成
```

**Step 4：深入（可选）**

用户可以选择：
- 深入某个概念 → 读概念页，展开完整机制
- 沿某条脉络延伸 → 给出脉络中相邻概念的编剧操作
- 转入编剧操作 → 从诊断结果进入模块二B
- 直接生成剧本 → 进入模块四

## 模块二B：编剧操作

从诊断结果或用户直接需求出发，给具体的写法建议，**同时标注俗套化风险**。

### 操作流程

**Step 1：确认操作方向**

- 如果从诊断转入：直接用诊断结果中的"盲区"和"深度不够"作为操作方向
- 如果用户直接请求：先快速扫描素材确认方向

**Step 2：生成操作建议**

每条建议的格式：

```
### [操作名]

**概念来源**：[概念名]（[脉络名]）

**写法**：[具体的写法指导，不是抽象原理，是"你在这个场景里可以这样做"的指令]

**经典用例**：[哪部电影用过，怎么用的]

**俗套化风险**：
- 风险信号：[出现这些特征说明你在滑向俗套]
- 避免方式：[怎么让它重新变新鲜]
```

**Step 3：场景改写（可选）**

用户要求时，对具体场景做改写示范。输出格式：

```
原文：[用户的原始素材]
→
改写版：[改写后的版本]
→
改动说明：[改了什么，为什么要这样改，附概念出处]
```

### 俗套化判断标准

以下信号出现时，标注俗套化风险：
- 同一条脉络只用最表层的概念，不往深处推
- 结构性套路在用但没有概念层的变体（如：用了"日常→裂痕→崩塌"但裂痕只靠 jump scare 制造）
- 角色反应是类型片标配（尖叫、逃跑、作死式开门）而非认知机制驱动的行为
- 威胁设计只依赖一个概念，没有跨脉络的组合
- 结局闭合方式是类型片默认选项（怪物被消灭/主角幸存）而非概念逻辑的必然结果

## 模块三：剧本生成

全流程生成恐怖片剧本。概念层作为参考背景自然融入，不搞"每场戏必须激活一个概念"的硬约束。

### 生成流程

**Step 0：确认篇幅和格式**

| 篇幅 | 时长/字数 | 格式 |
|------|----------|------|
| 长片 | 90-120分钟 | 标准编剧格式（场景标题+动作描述+对白） |
| 短片 | 10-30分钟 | 文学性剧本（重氛围描写和心理刻画） |
| 鬼故事 | 500-3000字 | 极简叙事（一个情境，一个转折，一个留白） |

用户没指定时，默认短片。

**Step 1：概念选配**

根据用户给的方向/素材/想法，从十三条脉络中选2-4条主线。

选配逻辑：
- 用户的想法天然靠近哪条脉络？→ 优先选
- 哪条脉络和主线互补？→ 选1-2条做副线
- 避免只选一条线——单脉络的剧本容易扁平
- 避免"所有脉络都用"——太多线会散

输出概念选配方案，问用户确认。

**Step 2：结构设计**

根据篇幅和概念选配，设计叙事结构。读 `references/patterns.md` 的结构性套路，选一个基础结构做骨架。

| 篇幅 | 结构要点 |
|------|---------|
| 长片 | 三幕结构：建置（日常→裂痕）→ 对抗（崩塌→认知失效）→ 解决（压抑回归/开放/崇高）。每幕30-40分钟 |
| 短片 | 单一情境+一次相变：日常→临界点→崩塌。10-30分钟 |
| 鬼故事 | 一个锚点+一个留白：正常情境→一件不对的事→蔡格尼克式不闭合。500-3000字 |

输出结构大纲，包含：幕/段划分、每段的核心概念、关键转折点、信息分配策略（观众>角色 / 角色>观众 / 无人知道）。问用户确认。

**Step 2.5：大纲确认**

结构确认后，输出完整大纲（每场戏的一句话摘要 + 视觉化检验），**等用户确认后再进入场景展开**。

大纲格式：

```
场景[N]：[地点] / [时间]
摘要：[一句话说这场戏发生了什么]
视觉化检验：[关键信息如何通过画面传达？如果无法视觉化，标注⚠️并给出替代方案]
概念背景：[这场戏自然激活的认知机制——仅标注，不展开]
```

视觉化检验原则：每个关键恐怖信息都必须是一帧画面能传达的，不需要旁白或内心独白补充。如果某个恐怖节点只能通过叙述传递（如"楼上的脚步声"），必须找到视觉替代（如"天花板上的震动"）。

**Step 3：场景展开**

逐场景展开。每个场景写：
- 场景目标（这场戏要让观众感到什么？）
- 概念机制（参考背景：这场戏天然激活了哪些认知机制？不强制但要有意识）
- 动作和对白
- 节奏标记（张力上升/持稳/释放）

**Step 3.5：场景检查点**

场景全部展开后、拼完整剧本之前，输出场景清单让用户确认：
- 列出每个场景的一句话摘要
- 标注张力曲线（示意整部片的节奏起伏）
- 问用户：「场景方向和节奏是否可以？需要调整哪个场景？」
- 用户确认后再进入 Step 4

**Step 4：完整剧本**

把所有场景组合成完整剧本，统一语言风格和节奏。

写完后做一轮自检：
1. 概念机制覆盖度：主线脉络是否都有至少一个场景深度激活？
2. 俗套化检查：角色反应是否类型片标配？威胁是否单概念？结局是否默认选项？
3. 节奏检查：张力曲线是否有起伏？是否有喘息空间？
4. 篇幅检查：字数/时长是否在目标范围内？

自检发现问题时直接修改，把修改记录附在剧本末尾。

**Step 5：迭代修改**

用户可以要求修改特定部分：
- 「这场戏不够恐怖」→ 回到模块二B，找盲区，改写
- 「这个角色太工具人」→ 用身体与认同线/异化线重新设计角色
- 「节奏太拖」→ 调整间歇强化的分布，加或减 jump scare
- 「结局太俗」→ 换一条脉络的闭合逻辑，或改开放结局

## 异常处理

| 场景 | 处理方式 |
|------|---------|
| 概念查询无匹配 | 列出最接近的3个概念供选择；如果确实不在映射中，建议用户用 concept-studio 补充概念 |
| 用户素材过少（一两句话） | 先追问细节（场景空间？角色关系？威胁类型？），素材不够不给诊断 |
| `references/concepts.md` 读取失败 | 降级为纯概念库查询：直接读概念页目录，但无法提供编剧操作映射 |
| concept-studio 不可用 | 概念补充流程暂停，告知用户需要 concept-studio 才能沉淀新概念 |
| 素材不属于恐怖片 | 坦诚说明超出工具箱范围，不强行用恐怖片框架解读 |
| 用户给出完整剧本 | 先做全局扫描（各幕各用到了哪些脉络），再逐场景诊断 |
| 生成的剧本俗套化 | 自检未通过时，回到 Step 2 换一条脉络的结构逻辑，或回到 Step 1 补充副线 |
| 用户要求改篇幅 | 长片↔短片：保留核心概念配置，重构结构；短片↔鬼故事：提炼核心情境，压缩到极简 |

## 约束

- 恐怖大师是 concept-studio 的下游消费者，概念页由 concept-studio 管理，恐怖大师不建独立概念库
- 概念页通用泛化，恐怖片只是现实锚点之一——恐怖片专用操作映射放在 `references/concepts.md`
- 不要做成"恐怖片编剧教程"——做成"认知工具箱"
- 诊断时给出概念出处，让用户能追溯操作建议的理论来源
- 如果用户给的素材不涉及恐怖片，不要强行用恐怖大师框架解读——坦诚说明这超出了工具箱的范围
- 编剧操作建议必须标注俗套化风险——每一个"你可以这样写"都要配套"什么时候这个写法会变俗套"
- 支持不同篇幅：长篇剧本、短片、鬼故事——篇幅不同，结构设计和操作颗粒度不同，但概念框架一致
- 概念层作为参考背景自然融入生成流程，不搞硬约束——生成时概念提供方向感，但写戏靠的是对情境和人物的理解
- 所有内容必须能视觉化呈现——每个关键信息都必须是一帧画面能传达的，不依赖旁白或内心独白。项目的最终目标是视觉化呈现（视频/动画），剧本就是分镜的蓝图
