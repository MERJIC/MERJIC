# 恐怖大师 · 认知驱动恐怖叙事工具

## 这是什么

一个基于认知科学概念的恐怖片/短篇/鬼故事生成工具。它不做"教你写恐怖片"的教程——它是一套**操作工程**：

- **concepts.md** 告诉你"为什么这个操作能制造恐惧"（理论层）
- **patterns/** 告诉你"别人用过什么模式、具体怎么操作"（方法层）
- **craft-checklist.md** 帮你检查"写完的东西有没有问题"（检验层）
- **主流程**把以上三者串起来，从你的想法/种子出发，一步步产出完整剧本

## 主流程

这是本 skill 的核心。所有调用最终都走这条线——区别只在于从哪一步进入。

```
Step 0  确认篇幅和格式
Step 1  概念选配（从十三条脉络选主线）
Step 2  结构设计（读 patterns/ 对应模块）
Step 3  大纲（每场戏摘要 + 视觉化检验）
Step 4  场景展开（逐场写动作和对白）
Step 5  自检（craft-checklist + 俗套检查 + 节奏验证）
Step 6  交付 + 沉淀判断（可选）
```

---

### Step 0：确认篇幅和格式


| 篇幅        | 时长/字数             | 格式             | 结构参考                                          |
| --------- | ----------------- | -------------- | --------------------------------------------- |
| 长片        | 90-120分钟          | 标准编剧格式         | `pacing.md` 三幕 + `information.md` 幕间切换        |
| 短片        | 10-30分钟           | 文学性剧本          | `pacing.md` 单一相变 + `character.md` 开篇结尾        |
| **5分钟短片** | **~1500字 / 5页剧本** | **极简 beat 结构** | `**information.md` 反转类型 + `pacing.md` 5分钟网格** |
| 鬼故事       | 500-3000字         | 极简叙事           | `information.md` 延迟揭露 + `character.md` 结尾策略   |


用户没指定时，默认短片。

---

### Step 0.5：灵感启动（可选）

用户没有给出具体点子时，从素材池提取种子：

1. **5分钟/鬼故事** → 从 `reddit/twosentence.md` 按知情权模式筛选或随机抽一条做种子
2. **中等篇幅** → 从 `reddit/nosleep.md` top 30 的开头分析开篇模式做起点
3. 有具体方向但需要填充 → 跳过，直接进 Step 1

确定种子后告诉用户方向选项，等确认再往下走。

---

### Step 1：概念选配

根据用户给的方向/素材/想法，从 concepts.md 的十三条脉络中选 **2-4 条主线**。

选配逻辑：

- 用户的想法天然靠近哪条脉络？→ 优先选
- 哪条跟主线互补？→ 选 1-2 条做副线
- 避免只选一条（扁平）和选太多（散）

输出选配方案，**等用户确认后再进 Step 2**。

---

### Step 2：结构设计

按路由索引定位到具体模块文件来设计：


| 设计任务  | 读哪个文件                     | 输出什么                 |
| ----- | ------------------------- | -------------------- |
| 节奏骨架  | `patterns/pacing.md`      | 硬停点位置、节奏形态选择         |
| 信息操作  | `patterns/information.md` | 知情权指纹（基础位置+动力学）、反转类型 |
| 角色/威胁 | `patterns/character.md`   | 主角设定、威胁本质、主角驱动检验     |
| 开篇/结尾 | `patterns/character.md`   | 开头策略、结尾方向            |
| 元素填充  | `patterns/elements.md`    | 场景空间、身体恐怖类型、社会背景     |


**篇幅对应的结构模板：**


| 篇幅        | 结构要点                                                       |
| --------- | ---------------------------------------------------------- |
| 长片        | 三幕：建置（日常→裂痕）→ 对抗（崩塌→认知失效）→ 解决（压抑回归/开放/崇高）。每幕 30-40min      |
| 短片        | 单一情境+一次相变：日常→临界点→崩塌。10-30min                               |
| **5分钟短片** | **3-5 个 beat，每个 = 动作+反应+意义转变。选一种信息不对称主打。整部片子 = 一个 Teaser** |
| 鬼故事       | 一个锚点+一个留白：正常→不对劲→不闭合                                       |


输出结构大纲（幕/段划分、核心概念、转折点、信息分配策略），**等用户确认后进 Step 3**。

---

### Step 3：大纲

结构确认后输出完整大纲：

```
场景[N]：[地点] / [时间]
摘要：[一句话]
视觉化检验：[关键信息能否用一帧画面传达？不能则标注 ⚠️]
概念背景：[这场激活的认知机制，仅标注]
```

**视觉化检验原则**：每个关键恐怖信息必须能被一帧画面传达。纯听觉/心理层面的恐怖必须找到视觉替代（"脚步声"→"天花板震动"）。

输出全部场景摘要 + 张力曲线示意，**等用户确认后进 Step 4**。

---

### Step 4：场景展开

逐场写：

- 场景目标（要让观众感到什么？）
- 动作和对白
- 节奏标记（张力↑ / 持稳 / 释放↓）

概念机制作为参考背景自然融入，不强求每场激活某个概念，但要有意识。

---

### Step 5：自检

写完后逐项过：

1. **概念覆盖度**：主线脉络是否都有至少一个场景深度激活？
2. **俗套检查**：角色反应是否类型标配？威胁是否单概念？结局是否默认选项？（对照 `tropes-anti.md`）
3. **节奏检查**：张力曲线是否有起伏？（对照 `pacing.md` 的节奏形态）
4. **篇幅检查**：字数/时长是否在目标范围？
5. **Craft 自检**：读 `craft-checklist.md`，逐条过开放性问题集

发现问题直接改，修改记录附在剧本末尾。

---

### Step 6：交付 + 沉淀判断

交付剧本给用户。然后执行沉淀扫描（**可选，每次问用户**）：

#### 6a. 片段沉淀

扫描生成内容，找出独立可复用的组件：

- 特别好的 beat → 可追加到 `patterns/information.md` 对应反转类型的示例区
- 有效的开篇段落 → 可追加到 `patterns/character.md`
- 好的结尾设计 → 可追加到 `patterns/character.md`

问用户：「这个片段我觉得不错，要存到参考库里吗？」

#### 6b. 组合记录

这次用的概念组合以前用过吗？没有则记录到 `references/generated-combinations.md`：

```markdown
## [日期] [篇幅] [题材关键词]

**组合**：[基础位置] + [动力学/反转类型] + [节奏形态]
**概念主线**：[用了哪些脉络]
**用户反馈**：[有效/一般/需调整]
**备注**：[什么效果好、什么不行、下次怎么调]
```

#### 6c. 用户原创点子

用户在使用过程中自己贡献的点子/设定，记录到 `references/generated-ideas.md`：

```markdown
## [日期]

**原始想法**：[用户的原话或描述]
**展开方向**：[实际走了哪个方向]
**用到哪些概念**：[脉络列表]
**状态**：已使用 / 待展开 / 存档备用
```

## 约束

- 概念层是方向感，不是硬约束——生成时概念提供指引，写戏靠对情境和人物的理解
- 所有内容必须能视觉化呈现——最终目标是视频/动画，剧本是分镜蓝图
- 编剧操作建议必须标注俗套化风险——每个"可以这样写"配套"什么时候会变俗套"
- 概念补充和沉淀由 concept-studio 负责，本 skill 不建独立概念库
- 如果素材不属于恐怖片，不强行套框架——坦诚说明超出范围
- **沉淀永远可选、永远经用户确认**——不自动存储任何生成内容
